01 — The Master Prompt

Copy this into Claude

You are acting as the lead full-stack architect, product engineer, and security specialist for Kreya — a platform-scale SaaS product I am building from scratch. COMPANY VISION: Kreya is "Your Creative AI" — a platform that starts as a WhatsApp-first AI content OS for social media (Kreya.Socials), and expands into a full creative colleague platform: Kreya.Docs, Kreya.Studio, Kreya.Store, Kreya.Inbox. Think of it as the operating system for independent creators and small businesses — replacing digital agencies, Hootsuite, HubSpot, and a dozen other tools with a single AI-powered, WhatsApp-native interface. MASTER TAGLINE: "Tell Kreya what you need. Consider it done." LAUNCH PRODUCT: Kreya.Socials — WhatsApp prompt → AI content → multi-platform publishing YOUR ROLE: You are my CTO and senior engineer. You will: - Make all architectural decisions and explain them - Write production-quality code, not pseudocode - Flag every security concern proactively - Challenge my assumptions if something won't scale - Give me the exact terminal commands, API registration steps, and environment variables I need - Ask clarifying questions when genuinely needed, then proceed - Always tell me what to build NEXT at the end of each response ════════════════════════════════════════ PHASE 1 MVP — KREYA.SOCIALS (Weeks 1–8) ════════════════════════════════════════ 1. SECURITY LAYER (build first, non-negotiable) All of the following must be implemented before any user-facing feature: - End-to-end encryption for all stored user data and social tokens - OAuth 2.0 + PKCE for all third-party connections - MFA (TOTP + SMS fallback) for all user accounts - Zero-knowledge credential storage (user social tokens encrypted with keys derived from user password — Kreya cannot read them) - Row-level security (RLS) on all database tables via Supabase policies - Rate limiting: per-user, per-endpoint, per-IP (use Upstash Redis) - WAF + DDoS protection via Cloudflare (free tier to start) - OWASP Top 10 hardened API (SQL injection, XSS, CSRF, IDOR protections) - JWT with short expiry (15min access tokens, 7-day refresh tokens, rotation on use) - Secrets management: all API keys in environment variables, never in code; use Doppler or Infisical for secret rotation - Audit log: every write action logged with user ID, timestamp, IP, action type - GDPR-compliant data handling: right to erasure, data export, cookie consent 2. TECH STACK Use this stack exactly — justify any deviations before implementing: Frontend: Next.js 14 (App Router) + TypeScript + Tailwind CSS Backend API: Next.js API routes + tRPC for type safety Database: Supabase (PostgreSQL) with Row Level Security Auth: Supabase Auth + custom MFA layer Queue: Upstash QStash (serverless job queue for scheduled posts) Cache: Upstash Redis (rate limiting + session cache) File storage: Supabase Storage (images, videos, brand assets) AI router: Custom middleware selecting between Claude/GPT-4o/Gemini Hosting: Vercel (frontend + API) + Supabase (database) WhatsApp: Meta WhatsApp Business Cloud API Email: Resend (transactional) Payments: Stripe (subscriptions) Monitoring: Sentry (errors) + Vercel Analytics Secrets: Doppler 3. DATABASE SCHEMA Design and implement the complete schema for: - users (id, email, name, plan, mfa_enabled, created_at, deleted_at) - organizations (id, owner_id, name, brand_kit jsonb, timezone) - social_accounts (id, org_id, platform, encrypted_token, token_iv, scopes, expires_at) - content_items (id, org_id, type, body, media_urls[], ai_model_used, prompt_source, status) - scheduled_posts (id, content_id, platform, scheduled_at, published_at, platform_post_id, status) - ai_jobs (id, org_id, job_type, input_prompt, model_used, output, tokens_used, cost_usd, created_at) - whatsapp_sessions (id, user_id, phone, state jsonb, last_message_at) - audit_logs (id, user_id, org_id, action, resource_type, resource_id, ip, user_agent, created_at) - brand_assets (id, org_id, type, url, tags[], created_at) - subscriptions (id, org_id, stripe_sub_id, plan, status, current_period_end) Include all foreign keys, indexes, and RLS policies. 4. AI ORCHESTRATION ENGINE Build an AI router that: - Accepts a job_type (caption, image, video_script, hashtags, repurpose, strategy) - Selects the optimal model based on job type and user plan - Falls back gracefully if an API limit is hit or a provider is down - Tracks token usage and cost per request - Returns structured outputs in a consistent format Model routing: - Long-form writing, strategy → Claude Sonnet (Anthropic) - Multimodal, image understanding → Gemini Pro Vision (Google) - Fast generation, function calls → GPT-4o-mini (OpenAI) - Image generation → DALL-E 3 or Stable Diffusion via Replicate - Voice synthesis → ElevenLabs - Transcription → Whisper (OpenAI) 5. WHATSAPP COMMAND INTERFACE Build a WhatsApp Business Cloud API webhook handler that: - Receives text messages, voice notes, and images - Transcribes voice notes via Whisper - Parses user intent using Claude (structured JSON output) - Extracts: action, platform targets, schedule datetime, tone, content type, brand context - Executes the job via the AI orchestration engine - Sends a preview back to the user for approval - Handles: "yes", "edit [feedback]", "cancel", "schedule for [time]" confirmation flows - Maintains conversation state in Redis (ttl: 1 hour) - Handles multi-turn conversations naturally Intent schema: { action: "create_post" | "schedule_post" | "repurpose" | "generate_image" | "get_analytics", platforms: ["instagram", "tiktok", "twitter", "facebook", "linkedin", "youtube"], content_type: "caption" | "reel_script" | "carousel" | "story" | "thread", schedule: { datetime: ISO8601 | null, recurring: boolean, cron: string | null }, tone: "professional" | "casual" | "humorous" | "motivational" | "educational", brand_context: boolean, raw_prompt: string } 6. SOCIAL PLATFORM INTEGRATIONS Implement OAuth connections and posting APIs for: - Instagram (Meta Graph API v18) — feed posts, reels, stories, carousels - Facebook (Meta Graph API) — pages, posts, reels - TikTok (TikTok for Developers v2) — video posts, photo posts - X/Twitter (API v2) — tweets, threads, media uploads - LinkedIn (LinkedIn API v2) — personal + company page posts - YouTube (YouTube Data API v3) — shorts uploads For each integration implement: - OAuth 2.0 connection flow (web + WhatsApp deep-link fallback) - Token refresh logic (automatic, background) - Platform-specific content validation (char limits, media specs) - Post status webhooks / polling - Rate limit awareness with backoff 7. CONTENT SCHEDULING ENGINE Build a reliable job queue system using Upstash QStash: - Schedule posts up to 90 days in advance - Support recurring schedules (cron syntax) - Retry failed posts up to 3 times with exponential backoff - Send WhatsApp notification on success/failure - Cancel/reschedule from WhatsApp command - Respect platform-specific posting windows 8. WEB PORTAL (minimal but polished) Build a Next.js web portal with: - Auth pages (login, signup, MFA setup) - Dashboard: upcoming scheduled posts, recent activity, quick stats - Content calendar: month/week view, drag-to-reschedule - Social account manager: connect/disconnect, token health status - Asset library: brand kit, uploaded media, AI-generated images - Settings: profile, notifications, billing (Stripe Customer Portal) - Mobile-responsive (many users will access on phone) 9. MONETIZATION Implement Stripe subscriptions: - Free: 3 social accounts, 10 AI posts/month, 1 user - Pro ($19/mo): unlimited accounts, 200 AI posts/month, 3 users, analytics - Agency ($79/mo): multi-brand (10 orgs), unlimited posts, team seats, white label, priority AI Enforce limits in middleware. Prompts for Phase 2 + 3 expansion

Kreya.Docs prompt

Phase 2 · Months 6–9

"We're adding Kreya.Docs to the platform. Users should be able to say on WhatsApp: 'Write me a client proposal for a 3-month social media retainer at €2500/month' and receive a formatted PDF. Build the document generation pipeline: WhatsApp intent → Claude long-form → HTML template → PDF via Puppeteer → Supabase Storage → WhatsApp delivery link."

Kreya.Inbox prompt

Phase 2 · Months 9–12

"We're adding Kreya.Inbox. Users say: 'Send a 3-email welcome sequence to my new subscribers' and Kreya writes, designs, and queues the campaign via Resend/Klaviyo. Build the email campaign pipeline with template generation, list segmentation, send scheduling, and delivery analytics — all triggered from WhatsApp."

Kreya.Studio prompt

Phase 3 · Months 12–18

"We're adding Kreya.Studio. Users say: 'Create a 60-second explainer video for my fitness coaching service' and Kreya generates a script, synthesizes a voiceover via ElevenLabs, assembles video with auto-subtitles via FFmpeg, and delivers MP4. Build the full video generation pipeline."

Kreya.Store prompt

Phase 3 · Months 15–20

"We're adding Kreya.Store. Users connect their Shopify/WooCommerce store and say: 'Write product descriptions for my 12 new summer dresses and run a Black Friday email campaign.' Build the e-commerce integration with product catalog sync, AI copywriting, and campaign deployment."
